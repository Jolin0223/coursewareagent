# AIGC 互动课件 — 产品需求文档（PRD）

## 一、产品概述

AIGC 互动课件是一款面向教师的 AI 课件生成工具，用户只需输入一句话描述教学需求，系统即可自动生成可交互的 HTML5 教学游戏课件。产品支持独立使用，也可作为插件嵌入第三方课件编辑器。

### 1.1 核心价值

- 一句话生成：降低互动课件制作门槛，无需编程能力
- AI 需求理解：自动将自然语言拆解为结构化需求框架，用户可微调后确认
- 全流程可视：从需求分析到课件生成，每个阶段进度实时可见
- 灵感复用：通过灵感大厅浏览他人作品，一键同款快速二次创作

### 1.2 两种运行模式

| 模式 | 说明 |
|------|------|
| standalone（独立模式） | 完整的独立应用，包含侧边栏导航、对话生成页、我的作品页 |
| embedded（嵌入模式） | 作为课件编辑器的插件运行，左侧为编辑器画布，右侧通过抽屉面板承载完整功能，支持将生成的课件直接插入编辑器 |

模式切换入口：侧边栏底部的模式切换按钮（独立模式/嵌入模式图标切换）。

## 二、全局架构

### 2.1 路由结构

| 路由 | 页面 | 说明 |
|------|------|------|
| `/` | GeneratorPage | 对话生成页（首页），包含欢迎页、灵感大厅、对话交互区 |
| `/library` | LibraryPage | 我的作品页，管理已创建和收藏的课件 |

### 2.2 页面布局

**独立模式（standalone）：**
- 左侧固定侧边栏（可折叠） + 右侧主内容区
- 主内容区根据路由渲染对应页面

**嵌入模式（embedded）：**
- 全屏为编辑器页面（EditorPage）
- 点击编辑器区域或右上角按钮可打开右侧抽屉（EditorDrawer）
- 抽屉内承载完整的侧边栏 + 主内容区布局

### 2.3 侧边栏

侧边栏支持展开（280px）和折叠（64px）两种状态。

**导航项：**
- 新建任务（主按钮）：点击后清空当前会话，回到首页欢迎状态
- 我的作品：导航到 `/library`

**会话历史列表：**
- 展示在导航项下方，占据侧边栏剩余空间
- 详见「三、会话管理」

**底部区域：**
- 模式切换按钮组（独立模式 / 嵌入模式）
- 折叠时仅显示图标

## 三、会话管理

### 3.1 会话列表

位于侧边栏中，展示所有历史会话。

**排序规则：**
- 置顶会话优先展示
- 同级别按创建时间倒序

**搜索：**
- 点击搜索图标展开搜索框
- 实时过滤匹配标题的会话
- 再次点击或清空搜索内容可收起搜索框

### 3.2 会话操作

每个会话项 hover 时显示更多操作按钮（三点图标），点击弹出操作菜单：

| 操作 | 说明 |
|------|------|
| 置顶/取消置顶 | 切换会话的置顶状态，置顶会话显示置顶图标 |
| 重命名 | 进入行内编辑模式，回车或失焦确认，支持 ESC 取消 |
| 删除 | 直接删除会话，如果删除的是当前激活会话则清空激活状态 |

### 3.3 会话创建

会话在以下场景自动创建：
- 用户在欢迎页输入框发送第一条消息时
- 点击「一键同款」时（创建标题为"同款-{原课件名}"的会话）

会话标题规则：
- 普通会话：取用户首条消息的前 20 个字符
- 同款会话：`同款-{原课件标题}`

## 四、对话生成流程

这是产品的核心流程，分为以下阶段：

### 4.1 阶段总览

```
input → analyzing → loading-framework → framework → generating → completed
```

| 阶段 | 说明 |
|------|------|
| input | 初始状态，等待用户输入 |
| analyzing | 用户发送消息后，展示分析动画（旋转加载 + 文案提示） |
| loading-framework | AI 回复文案后，展示三点跳动动画，等待需求框架生成 |
| framework | 需求框架卡片流式输出中/已完成，等待用户确认 |
| generating | 课件生成中，展示四阶段进度面板 |
| completed | 生成完成，展示课件结果卡片 |

### 4.2 input 阶段 — 欢迎页

当无激活会话或会话无消息时，展示欢迎页：
- 标题："欢迎使用 AIGC 互动课件"
- 副标题："一句话生成教学互动游戏，让课堂更精彩"
- 居中输入框
- 下方灵感大厅区域（详见「八、灵感大厅」）

### 4.3 analyzing 阶段

用户发送消息后立即进入：
- 展示居中的旋转加载动画
- 主文案："正在分析您的需求..."
- 副文案："AI 正在理解您的教学目标和互动需求"
- 持续 1.5 秒后自动进入下一阶段

### 4.4 loading-framework 阶段

- AI 发送一条文本消息："我已理解您的需求，正在为您生成需求确认框架。"
- 消息下方展示三点跳动加载动画
- 持续 20 秒后自动展示需求框架卡片

**底部操作按钮：**
- 「跳过确认，直接生成」：跳过需求框架确认，直接进入生成阶段

### 4.5 framework 阶段 — 需求框架卡片

AI 生成结构化的需求框架，以卡片形式流式输出，包含三个区块：

| 区块 | 说明 |
|------|------|
| 用户需求 | 对用户原始需求的理解和总结 |
| 功能设计说明 | 课件的具体功能和交互设计 |
| 设计风格 | 视觉风格和动效描述 |

**流式输出逻辑：**
- 三个区块按顺序逐字输出，每个区块完成后间隔 400ms 开始下一个
- 输出过程中显示闪烁光标
- 普通生成流式时长：15 秒
- 一键同款流式时长：2 秒

**流式完成后：**
- 每个区块变为可编辑的 textarea
- 用户可自由修改内容
- 编辑后鼠标失焦自动保存，toast 提示"保存成功~"

**底部操作按钮：**
- 流式未完成时：「跳过确认，直接生成」
- 流式完成后：「确认需求，立即生成」

### 4.6 generating 阶段 — 四阶段生成

用户确认需求后进入生成阶段，展示进度面板，包含四个串行阶段：

| 阶段 | 说明 |
|------|------|
| 图片生成 | 生成课件所需的图片素材 |
| 代码生成 | 生成 HTML/CSS/JS 课件代码 |
| 代码审查 | AI 审查生成的代码质量 |
| 代码修复 | 修复审查中发现的问题 |

每个阶段有三种状态：pending（等待中）、in-progress（进行中，显示进度百分比）、completed（已完成）。

**图片生成阶段特殊逻辑：**
- 展示图片生成网格，每张图片有用途说明
- 图片逐张生成，显示加载动画后展示缩略图

**停止生成：**
- 生成过程中底部显示「停止」按钮
- 点击后中断生成，AI 发送"系统已暂停回答~"，回到 input 阶段

### 4.7 completed 阶段 — 课件结果

生成完成后：
- 展示课件结果卡片（详见「五、课件结果卡片」）
- 自动折叠侧边栏
- 自动打开右侧预览面板，展示生成的课件

## 十、一键同款

允许用户基于已有课件快速创建同款，预填充需求框架后微调即可重新生成。

### 10.1 触发入口

一键同款共有三个触发入口：

| 入口 | 位置 | 触发方式 |
|------|------|----------|
| 课件结果卡片 | 对话页生成结果 | 直接调用 store |
| 我的作品列表 | LibraryPage 课件卡片 | 直接调用 store |
| 预览窗口 | 独立预览窗口 | postMessage 通知主窗口 |

### 10.2 执行逻辑

1. 获取源课件关联的需求框架（RequirementFramework）
   - 通过 coursewareId 找到对应的会话
   - 从会话消息中提取 `requirement-framework` 类型的消息内容
   - 如果找不到对应会话，使用课件标题自动生成默认框架

2. 创建 clone 会话
   - 会话标题：`同款-{原课件标题}`
   - 预填充三条消息：
     - 用户消息（text）：`一键同款：{原课件标题}`
     - AI 消息（text）：`正在为您拉取「{原课件标题}」的需求框架，您可调整后点击 确认需求，立即生成。`（流式输出）
     - AI 消息（requirement-framework）：源课件的需求框架（流式输出，持续 2 秒）

3. 导航到首页，激活新会话

4. 页面自动进入 framework 阶段
   - AI 引导文案流式输出
   - 需求框架卡片以 2 秒时长流式输出
   - 流式完成后，框架变为可编辑状态
   - 底部显示「确认需求，立即生成」按钮

5. 用户微调框架后点击确认，进入正常的生成流程

## 十二、嵌入模式

产品作为插件嵌入第三方课件编辑器时的完整交互逻辑。

### 12.1 编辑器页面（EditorPage）

嵌入模式下的主页面，模拟课件编辑器画布。

**状态与展示：**

| 状态 | 背景图 | 点击行为 |
|------|--------|----------|
| 未插入课件 | 编辑器概览图 | 打开右侧抽屉 |
| 已插入课件，无更新 | 编辑器含课件图 | 打开右侧抽屉 |
| 已插入课件，有版本更新 | 编辑器含更新提示图 | 同步到最新版本 |

**右上角：** 模式切换按钮（可切回独立模式）

### 12.2 右侧抽屉（EditorDrawer）

- 从右侧滑入，宽度 75vw（最大 1200px）
- 有半透明遮罩层，点击遮罩关闭
- 顶部标题栏："AI互动课件" + 关闭按钮
- 内部承载完整的 MainLayout（侧边栏 + 主内容区），标记为 embedded 模式

### 12.3 插入课件

在嵌入模式下，以下位置会显示「插入课件」按钮：
- 课件结果卡片
- 灵感大厅课件卡片
- 我的作品课件卡片
- 预览面板工具栏

**插入逻辑：**
- 将课件数据快照存入 `insertedCoursewares` 列表
- 存储字段：id、title、version、htmlContent、slideIndex
- 插入后自动关闭抽屉
- toast 提示：`"{课件标题}" 已插入课件`

### 12.4 版本更新通知

课件插入编辑器后，模拟版本更新机制：

1. 插入 8 秒后，自动触发版本更新通知
2. 编辑器页面切换为"有更新"状态（更换背景图，显示更新提示）
3. toast 提示："互动课件有新版本可用，请查看编辑器"
4. 用户点击编辑器区域执行同步
5. 同步后课件版本更新为最新版本，toast 提示"课件已同步至最新版本~"

### 12.5 源课件下架处理

当已插入编辑器的课件被源作者删除时：

**触发条件：** 用户在"我的作品"中删除一个已被插入编辑器的课件

**处理逻辑：**
1. 课件从主列表中删除
2. 编辑器中已插入的课件标记为 `isSourceDeleted = true`
3. 同时清除该课件的 `hasUpdate` 状态

**编辑器展示：**
- 底部居中显示下架提示横幅
- 提示文案："源互动课件已被作者下架，当前为缓存版本，无法同步更新"
- 提供「移除课件」按钮
- 点击移除后从已插入列表中删除，toast 提示"已移除下架课件"

**禁用逻辑：**
- 已下架课件不再触发 8 秒版本更新定时器
- 无法执行版本同步操作

## 十三、输入框

对话页底部的消息输入组件。

### 13.1 基本功能

- 多行文本输入（textarea），自动调整高度
- 最大高度限制，超出后出现滚动条
- 发送按钮：点击或 Enter 发送（Shift+Enter 换行）
- 空内容时发送按钮禁用

### 13.2 状态变化

| 状态 | 说明 |
|------|------|
| 正常 | 可输入，可发送 |
| 生成中 | 输入框禁用，显示「停止」按钮替代发送按钮 |
| 居中模式 | 欢迎页时输入框居中展示，发送后切换为底部吸附模式 |

### 13.3 嵌入模式差异

嵌入模式下输入框右侧额外显示课件关联数量角标，展示当前编辑器中已插入的课件数量。
